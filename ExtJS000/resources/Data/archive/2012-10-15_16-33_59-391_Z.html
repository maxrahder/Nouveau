Ext.create('Ext.button.Button', {
    text: 'Button',
    renderTo : Ext.getBody() 
});

Introduction

This lab will help you discover how we will do some of the labs that use the Live Preview feature. Obviously, we haven't covered any concepts or syntax yet, so don't worry about the details for now.

Live Preview labs are intended to be small, stand-alone labs that let you try out syntax or a few features.

Important: For any Live Preview lab, just like Live Preview examples in the API documentation, if you leave the page you lose your work! You can open new windows or tabs, but be careful not to leave the lab instructions page until you've finished.

Steps

  1. See the code at the top?

    Some labs launch with a panel at the top of the page containing a code snippet. When that happens, lab steps will have you preview, modify and run the code. Other labs have you use a text editor or your IDE to modify code.

  2. Try it out

    Click "Live Preview" button and the code is run. Clicking back on "Code Editor" goes back to showing you the code.

  3. Examine the code

    The code runs a function, Ext.create(), passing two arguments.

    The first argument is the name of the class being created — either a class in the Ext JS library, or a class you've defined. In this case, it's the name of the button component.

    The second argument is a JSON object, describing the thing being created. The button is being created with two properties: text and renderTo, described below.

  4. What is renderTo?

    Right now, we don't really care. ;-)

    The renderTo property specifies where in the DOM the component creates itself. You'll rarely use renderTo, unless you are coding a little example or test, like we are here.

    We will discuss renderTo in more detail later in the class.

  5. Change the button's text

    Click on Code Editor and change the text property to some other string. Then press Live Preview. You should see your changes. The API documentation, lecture slides and labs have numerous Live Preview examples.

    Seeing examples and trying things out is one of the best ways to learn Ext JS.

  6. Add a click event handler

    As long as we're trying things we haven't covered in lecture, let's try to add some code to be run when you click on the button.

    For buttons, the handler option specifies a callback function to be run when the user clicks on it.

    Here's an example.

    handler: function(button){
	alert('You clicked the ' + button.getText() + ' button!');
}

    Add a handler, like the one above, and try it out. If you have trouble getting it to work, ask your instructor to help.

  7. Give the button sub-items

    Buttons have a lot of properties. For example, they can be made "toggleable," they can have an icon and they can have a set of sub-items. Let's add sub-items under the button.

    To do that, buttons have a menu config property that can take an array of button configs. Here's an example:

    menu : [{text: 'Option One'},{text: 'Option Two'}]

    Add that config property to your code.

    Notice that these sub-items are just JSON objects with a text property. In the case of buttons, it is assumed that the menu array items are button configs.

    Try it out

    Press Live Preview and see how it looks. Note that as you click on your button, your event handler is still run, and the dropdown of sub-items is shown.

  8. Use a split button instead

    Rather than have the handler run when all you want to do is show the sub-items, there's another type of button called a split button.

    Ext.button.Split subclasses Ext.button.Button. It's just a specialized kind of button.

    To use a split button, change the first parameter being passed to Ext.create() to reference the split button class instead. Run it. Now, the drop down icon only shows the sub-items. Clicking elsewhere on the parent button runs its handler.

    9. Optional

  9. Add more handlers, and more sub-items

    Since each sub-item in the menu array is the config of a button, you can add a handler and menu arrays to those too. Add a few buttons under one of the sub-items. Put a handler on one of those.

    A word of caution As JSON objects get more complex it's easy to forget a comma, or get confused about the bounds of a nested object or array. So it's best to make very small changes, then run the code to make sure everything still works. Then add another small change, and test that. And so on.

Conclusion

The purpose of the lab was for you to discover the benefits of Live Preview and get a feel for how we will be doing many of our labs. You also saw how to create buttons, add a handler, add sub-items and use a split button.
