Introduction

In this lab, you will start coding the YelpExtplorer application. This week we will build on the application until the complete application is coded, themed and deployed.

  1. Generate a workspace

    Open a terminal window and change directories to your Ext JS directory.

    Generate a new workspace by typing

    sencha generate workspace ../SenchaTraining/ExtJS251/Labs

    Inspect the SenchaTraining/ExtJS251/Labs folder and note that Sencha Cmd has created a new folder named ext. This is a copy of the essential libraries and resources needed to develop and run Ext JS applications.

    Furthermore, all applications coded under the Labs folder will share the ext install.

  2. Generate the YelpExtplorer application

    In the terminal window, navigate to SenchaTraining/ExtJS251/Labs/ext and generate the YelpExtplorer application by typing

    sencha generate app YelpExtplorer ../YelpExtplorer
  3. Run the application

    In your browser, open the initial application.

    You should see a single tab titled "Center Tab 1" in the center and a panel titled "west" on the left. The application doesn't do anything yet.

  4. Inspect the application

    In your IDE look at the SenchaTraining/ExtJS251/Labs/YelpExtplorer/ folder. You should see several files, organized according to Sencha's MVC file structure:

  5. Inspect index.html

    In your IDE open .../YelpExtplorer/index.html. Note that it references two JavaScript files: bootstrap.js and app.js.

    The bootstrap file contains classpath and alias mappings. Don't worry about the details of bootstrap.

    app.js is the starting point of any Ext JS application.

  6. Inspect app.js

    In your IDE open .../YelpExtplorer/app/app.js.

    app.js is the starting point of an Ext JS application. It is an instance of Ext.app.Application . This code specifies the appliation's controllers, and creates the viewport, which is the top-level container for the the user interface.

  7. Inspect Main.js

    In your IDE open .../YelpExtplorer/app/controller/Main.js.

    This file contains the definition for a controller. Note that it extends Ext.app.Controller . Controllers coordinate events between data stores and components, but our controller doesn't do anything yet.

  8. Inspect Viewport.js

    In your IDE open .../YelpExtplorer/app/Viewport.js.

    This defines a viewport, using the border layout. It contains two components: a west panel and a center tab panel. The tab panel contains a single panel whose title is "Center Tab 1"

Conclusion

In this lab, you used Sencha Cmd to generate a new workspace. A workspace lets multiple applications share resources. You also generated an application, and reviewed the initial MVC file strucure.

As we write YelpExtplorer we will be refactoring this starting point, adding new views models, stores and controllers.
