So as a developer, you're sort of in two modes when writing code:

When defining classes you'll be in the role of the class designer, defining re-usable, decoupled components and classes.

Your classes will typically subclass components, stores and record classes from the Ext JS library.

You'll use your components within other components. Ultimately, those are assembled into the full user interface for your application.

Furthermore, you'll write the code that reacts to events, such as the user clicking on a Save button. That code will have an application-wide perspective — updating records, sending data to the servet, etc.
