Introduction

For this lab, you will need a working IDE (e.g. Eclipse) with VJET Sencha Plugin installed. If that has not been installed, please follow the setup instructions.

You can use SenchaTraining/extJS/labs/MVC/MusicLibrary/ but you don't have to save any code. The purpose of this lab is to show you how to setup a VJET for Ext JS project.

Steps

  1. Project setup in Eclipse with the Sencha Eclipse Plugin.
    1. Open Eclipse, create new VJET project
    2. Give the project the name MusicLibrary
    3. Select Create project from existing source and choose the folder Student/labs/MVC/MusicLibrary/ folder.
    4. Select Use project folder as root for sources
    5. Press Next
    6. Click the link: Configure inclusion and exclusion filters
    7. Add the following exclusion pattern: ext/

      This will tell the plugin not to error check framework files.

    8. Press Finish and select the Projects tab
    9. Click Add and check ExtJSTL-version
    10. Close the window and press Finish
  2. Check if the auto complete code works.
    1. Make sure Project > Build Automatically is enabled
    2. Open MusicLibrary/app/GridViewer.js

      This file is empty. Enter: Ext.def and press control + space. You should see a popup with possible code solutions.

    3. Use the arrow keys and select Ext.define(). Press enter.

      Use the following classname: 'MusicLibrary.view.Gridviewer'.

    4. Within the brackets enter: ex and press control + space.

      Choose the extend config and press enter.

    5. Enter ' ', and within the quotes press control + space

      Start typing: Grid, select GridPanel and press enter. It should extend from the class Ext.grid.Panel.

This gives you an idea how auto complete code in Eclipse with the Ext JS plugin works.

You can close this lab; you don't need to save the code.
