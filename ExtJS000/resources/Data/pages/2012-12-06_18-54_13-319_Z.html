Introduction

This lab will let you create and modify toolbars and docked items.

Steps

  1. Inspect Labs/Toolbars

    Use your IDE to inspect Labs/Toolbars. The code creates a simple panel.

  2. See how it looks

    Visit Labs/Toolbars in your browser. You should see the panel.

  3. Add a toolbar with one button

    Use the tbar property.

    Recall that tbar stands for "top toolbar". It is an array of the items to be in the toolbar. The default xtype for toolbars is button, so for the JSON object in the array simply specify its text property. Set the text to "One".

    Save and refresh your browser window, and you should see the toolbar and button at the top of the panel.

  4. Add a second toolbar

    Copy and paste the toolbar, so there are two identical toolbars. Don't even change the array name. In other words, you should have two identical lines, both specifying tbar.

    Preview in your browser and you'll just see a single toolbar. That reflects the nature of JSON. Since the configis a JSON object, with name-value pairs, there can only be a single value for a given name. So the second value overwrites the first.

    Change the second toolbar to use bbar, save and refresh. You should see two toolbars. Try using rbar and lbar too.

  5. Show two top toolbars

    As we saw, using a property twice doesn't work. So to have two items docked in the same place, you need to change the syntax to use the dockedItems array.

    Recall from lecture that dockedItems is an array of component configs, that specifies the xtype of the component, and its dock position: "top", "bottom", "left" or "right".

    Try it! In this case, you'll have two items in the array, each with xtype:"toolbar", and each with dock set to "top". Since they are toolbars, which are containers, they each have an items array, specifying their content. Put a button in each.

    In theory you can mix using dockedItems with the convenience properties tbar. Try it: use both dockedItems, with your two new top-docked toolbars, as well as tbar. Give each button a unique name so it's easy to tell one toolbar from another.

    Save and refresh, and you'll see that the tbar is added last. That's mostly just a curiosity: in practice you'll use one or the other. Either dockedItems, or the corresponding convenience property, like tbar. If you need to dock two things in the same location, then use dockedItems.
