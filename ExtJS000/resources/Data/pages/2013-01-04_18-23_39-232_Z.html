Some mixins are designed to be both used alone or as a mixin. Ext.util.Observable is one example.

  • If you simply need an observable object, you can either create it directly
  • If you need additional functionality, you can extend it
  • If you are already extending another class, and need the functionality, then use it as a mixin

The last case is tricky, since Ext.util.Observable has a constructor. We'll see an example in a few slides.
