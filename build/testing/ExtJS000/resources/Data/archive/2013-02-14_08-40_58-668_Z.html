There are a couple of factors that affect performance

This affects initial startup time heavily, but also data store load time.

The solution is to use the Sencha Cmd:

  • Build a single, concatenated JavaScript file containing all the required JavaScript used by the application
  • Build a minified CSS stylesheet that uses only mixins for the used components.

A CSS selector like .HeaderContainer .nav span will be processed, and an attempt made to match it on every span in the document. It’s much more efficient to use a single, identifying class name on elements to be styled.

There are several points to bear in mind when optimizing performance of JavaScript code:

  • Avoid older or badly written JavaScript engines.
  • Optimize code which is repeated frequently.
  • Optimize code which is executed at render or layout time.
  • Better still, try not to execute any extra code at initial render or layout time.
  • Move invariant expressions outside of loops
  • Use for rather than Ext.Array.each
  • If a function performs its task conditionally and is frequently called, check the condition outside the call, and only call it if necessary. (See calls to fireEvent in Ext JS codebase)
  • Setup and teardown of a call frame (apparatus needed to make a function call) is slow on bad JavaScript engines
It’s important to reduce container/component nesting to avoid unwanted layout runs and DOM reflows.
For example; when you have a TabPanel and you want to add a Grid, don't wrap it with an extra Panel. A grid already extends Ext.panel.Panel.
