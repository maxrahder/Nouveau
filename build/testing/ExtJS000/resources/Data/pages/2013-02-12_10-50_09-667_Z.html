Introduction

Creating the SenchaTraining folder structure with Sencha Cmd.

Before you can start with this lab, you must have a copy of the course files extracted in the root of your webserver (htdocs). If your setup is not complete, ask your instructor for assistance or read the instructions.

Objectives

  • Generate an Ext JS workspace

Steps

  1. Install Sencha Cmd 3.1.2.342

    In a terminal window, install Sencha Cmd 3.1.2.342 by typing the following: 

    sencha upgrade 3.1.2.342
    Note: If you don't already have a Sencha Cmd version installed, download and install it:
    OS X | Windows | Linux(32 bit) | Linux (64 bit)

  2. Verify your Sencha Cmd installation

    Close any open terminal windows, then open a new terminal and type the following: 

    sencha which
    If your Sencha Cmd version is newer than 3.1.2.342, navigate to the printed directory and run the "uninstall" script. Repeat this step until you are using version 3.1.2.342.

  3. Confirm you have the following folder structure in your htdocs folders:
    • SenchaTraining
      • ExtJS251
        • ext-4.2
        • Labs
        • LabSolutions
          • ext
  4. We will start by generating a workspace.

    Workspaces allow multiple applications to share the same Ext JS framework. Throughout the week you'll be working on multiple labs, and each lab is coded as its own application. By using a workspace, all the labs will share the same framework.

    Open your console/terminal and navigate to htdocs/SenchaTraining/ExtJS251/ext-4.2

    Run the following command.

    sencha generate workspace ../Labs
  5. Confirm that the workspace was generated correctly

    You will see 3 new folders in the Labs folder:

    • A .sencha hidden folder that stores the Sencha Cmd workspace configuration,
    • a new ext folder which contains the framework libraries that all of your applications will use, and
    • a packages folder which will contain themes, plugins and locales.

    Furthermore, all applications coded under the ExtJS251/Labs folder will share the ext install.

    You can test that it works, by opening one of the Labs in your browser.

Conclusion

In this lab, you used Sencha Cmd to generate a new workspace. A workspace lets multiple applications share resources. You also reviewed the initial MVC file structure and verified the generated workspace with a pre-existing application.
