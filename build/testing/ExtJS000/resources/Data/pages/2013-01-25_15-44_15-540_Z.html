Let's review what goes into each of those files and folders.

This is the starting point for an application. Ext.application() automatically creates an Ext.app.Application instance.

The application specifies the namespace.

Typically it auto generates a viewport.

The class in application.js extends Ext.app.Application.

It specifies the controllers used in the application.

Contains class definitions for the records used in the application. Each definition is a Ext.data.Model subclass.

Contains class definitions for the stores used in the application. Each definition is a Ext.data.Store subclass.

Contains class definitions for the components used in the application. Each definition is a Ext.Component subclass.

Note that these are the major reusable building blocks for your application. These will typically be container subclasses, like grids, tab panels, etc.

Controllers coordinate events between stores and views.
