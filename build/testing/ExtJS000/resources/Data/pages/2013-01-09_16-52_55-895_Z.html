Introduction

In this lab, you will use Sencha Architect to define Models and Stores for your Twitter search mobile application.

Steps

  1. In the upper-right corner of the Project Inspector, click [+] and choose Model.

  2. Select MyModel under Models.

  3. In the Config filter, start to type user. This will filter the user named properties.

  4. Select userClassName below and set the value to Tweet.

  5. In the Config filter, type in field.

  6. Click the [+] to the right of the Fields property and enter the following fields in a comma separated format: profile_image_url, from_user, text, created_at

  7. In the Toolbox filter, type in proxy.

  8. Drag the JsonP Proxy from the Toolbox and drop it onto Tweet in the Project Inspector.

  9. Choose the MyJsonPProxy and configure the following properties:

    • url: http://search.twitter.com/search.json

    • extraParams: {'q':'sencha'}

  10. In the Config filter, type in reader.

  11. In the reader property, click the [+] and choose the Json reader.

  12. Click the arrow to select the reader configs.

  13. In the Config filter, type in root.

  14. Select the rootProperty below and type in results.

  15. In the upper-right corner of the Project Inspector, click [+] and choose Store.

  16. Select MyStore under Stores.

  17. In the Config filter, start to type user into the filter field. This will filter the user named properties.

  18. Select userClassName below and set the value to Tweets.

  19. In the Config filter, type in store.

  20. Set the storeId config to Tweets.

  21. In the Config filter, type in model.

  22. Select model below, and set the value to Tweet

  23. In the Config filter, type in auto.

  24. Select the checkbox to the right of the autoLoad property.

  25. Click Save.

  26. In the Project Inspector, right click Tweets and select Load Data.

  27. In the Project Inspector, click the eye symbol to the right of Tweets and verify that data loaded.

  28. Save the project.
