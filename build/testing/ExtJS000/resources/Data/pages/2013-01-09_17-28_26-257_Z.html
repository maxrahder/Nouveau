Introduction

This is a group project. In groups, you will plan then implement a full-featured application from start to finish.

You will need to sharpen your pencils, get a fresh cup of coffee and have the API documentation ready.

Before getting started, your instructor will break you up into groups of two or three. You will work as a team to plan and decide on what you need to do. One person within the team will be the designated typest.

The lab has major steps, and for each step we will do this:

  • The instructor will pose a question or task
  • Within each group you will discuss the solution
  • The instructor will choose a group to explain their answer or plan to the rest of the class
  • The instructor will document the consensus on the white board
  • Each group will implement the solution

We will do this over and over as we implement each major piece of the application.

We will work in "Labs/Weather. It's empty.

Requirements

You've been hired by the US meteorological service, the National Oceanic and Atmospheric Administration (NOAA), to develop a Web application for displaying monthly weather data for cities around the world.

Your user sketched out what she wants:

Note these key components:

  • There's a tab panel, with a right-docked form
  • The tab panel contains a grid, a data view and a chart
  • Changes to the data are immediately reflected throughout the application
  • The grid, data view and form are kept in sync as the user selects items

Steps

  1. How will the application folder end up being structured: what folders, what files?

    If you use Sencha Cmd, how will you modify the application it generates to fit our requirements?

  2. Stub out the user interface

    Initially, make each of the major containers (the grid, view, chart and form) extend Ext.panel.Panel. Later we'll make them subclass the more specific types of panel, as needed.

    Each will need an xtype, and for now just use the html property to show what each will eventually contain.

  3. Write the model and store

    First, review the data feed and its URL — "data/parisTemperatureByMonth.json".

    Call the model Weather.model.Month and the store Weather.store.Months.

  4. Fully implement the grid
  5. Fully implement the data view

    Your CSS developer already wrote the CSS and the template that uses it. All you need to do is copy and paste from the code below. 

    .month { float: left; border: thin solid ; width: 180px; height: 90px; padding: 10px; margin: 10px 5px 0 5px; border: 1px dotted #999; background: #CCCCCC; color: #555; }
.month.over { background: #EEEEEE; }
.month.selected { background: white; border: 2px solid black; }

    Where does this CSS code need to be located.

    What dataview configs are used to reflect those CSS styles?

    Here's the template for use in the data view.

    <b>{month}</b><table><tr><td>High: </td><td>{high}</td><tr><td>Low: </td><td>{low}</td></tr><tr><td>Precipitation: </td><td>{precipitation}</td></tr></table>
  6. Have row selections be reflected in the data view, and selections in the data view be reflected in the grid
  7. Fully implement the form

    First code the user interface. Then add the code to have the form reflect the currently-select record. When that works, add the two following features — both will take some thought and planning.

    • Changes to fields are immediately reflected in the data
    • Optionally include a toolbar, with buttons to go to the previous and next record
  8. Fully implement the chart

    Get each series working before moving on to the next. Charting precipitation (using the right axes) is optional.
