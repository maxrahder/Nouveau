• Since you probably control your own data feeds, the type:'auto' may work for most of your data
  • But you can never use auto for dates, since they are sent as strings
  • For dates you must always specify type:'date' and you must provide a dateFormat that matches the data

Note that if you omit (or mis-spell) dateFormat, Ext JS will use the JavaScript Data.parse() method, which tries to convert the string as well as it can. Date.parse() is particularly bad at handling timezones. Therefore, you should always use dateFormat. See the API documentation for more information.
