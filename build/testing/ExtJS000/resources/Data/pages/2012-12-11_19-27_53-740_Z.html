But in OO programming, properties aren't used very much, because it's better to use accessor methods; we'd rather have programmers run getName() or setName() instead of using the property directly. One benefit is that if we have a setter we can add a business rule or fire an event as the setter is run.

To automatically generate accessor methods, use the config property. 

Ext.define('Mammal', {
    config : {
        name : 'Anonymous',
    }
});
var a = Ext.create('Mammal');

alert(a.getName()); // Alerts 'Anonymous'
a.setName('Roger');
alert(a.getName()); // Alerts 'Roger'
