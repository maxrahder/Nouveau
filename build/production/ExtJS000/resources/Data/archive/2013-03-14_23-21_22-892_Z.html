• In Ext JS 4.1, grid rows were rendered and discarded, one page at a time
  • In Ext JS 4.2 grids incrementally render rows during scrolling
  • The result is less latency and more fluid scrolling
  • In Ext JS 4.0 Sencha introduced "infinite scroll" grids, with somewhat complicated syntax involving pre-fetching data, buffering and code to specify the grid's scroll-bars.
  • In Ext JS 4.1 that was refactored, with data stores encapsulating buffering via buffered: true
  • In Ext JS 4.2 grid buffering is done via the Ext.grid.plugin.BufferedRenderer which decouples the grid from its backing store.

See the First Look at Ext JS 4.2 Grid blog post for more information.
