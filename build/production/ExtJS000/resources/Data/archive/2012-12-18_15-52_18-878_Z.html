UI's can be handy when you want to create a different set of styles within your theme. For example, you have a Green Neptune theme, but you may want some of your buttons red instead of green. You could create a "red" UI, and implement this ui online on the elements that should be red.

  • Some Ext components have CSS mixins that allow you to create new UI's.
    • Ext.panel.Panel
    • Ext.toolbar.Toolbar
    • Ext.window.Window
    • Ext.button.Button
  • These components in the Ext framework have a ui configuration property.

    Each of these mixins is named as: extjs-componentname-ui.
    You can find it in the API docs, which CSS Mixins are applied: For example: Toolbar

  • While UI mixins are a handy way to configure multiple appearances for a component, they should not be overused. Because each call to a UI mixin generates additional CSS rules, gratuitous calls to UI mixins can produce an overly large CSS file.
