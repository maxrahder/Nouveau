Introduction

This lab lets you experiment with border layout.

Steps

  1. Inspect Labs/Layouts/Border/app.js

    Use your IDE to inspect Labs/Layouts/Border/app.js . The code creates a window. Run the application using your browser. You should see a rectangular window, with the title "Border".

  2. Specify the window's layout and add two children

    Add a layout: 'border' property, and an items array. In the items array, specify two children using this code: 

    items : [{
    title : 'West'
}, {
    title : 'Center'
}]

    Save your changes and refresh your browser. It should not look very good. Why? Because we haven't specified the position of the regions.

  3. Add region properties

    To the child panel titled "West" add a region: 'west' property. Specify region: 'center' to the other panel.

    Save and refresh. Now the center panel takes up all the space. We still aren't seeing the west panel.

  4. Specify the heights and widths of the panels

    We aren't seeing the west panel because it needs to specify its width (or its children do).

    Add a width of 80 to the west panel, save and refresh. You should see the west panel.

    Try specifying a height of 10 to the west panel. And specify a height and width of 10 to the center panel. Save and refresh.

    Those settings are ignored. A panel on the sides takes up all the available height, so the height property is ignored. Panels on the top or bottom take up all the available width, so setting width on those panels is ignored. The center panel takes up all the remaining space, so both its height and width are ignored.

  5. Add other panels

    Add north, east and south panels. Set their heights or widths, where relevant.

    Save and refresh. Are you seeing all panels: north, east, south, west and center? Do they have the heights and widths you expect? If not, ask the instructor to help you figure out why.

  6. Set the north panel's relative weight

    By default, the north panel's weight makes it take up the entire width of the parent: the west and east panels are under it.

    Modify this priority by setting both the east and west panel weight to 20, and the north panel weight to 10.

    Save and refresh. Since the west and east panels have a greater weight than the north panel, they should both now extend to the top of the parent panel. The north panel should be between them.

  7. Make the panels collapsible

    Make the border panels collapsible via collapsible:true .

    This setting only affects the west, north, east and south panels. With border layout the center panel is always seen; it doesn't make sense to specify collapsible for the center panel.

    Save and refresh. Is each panel collapsible?

    Add collapseMode:'mini' to the west panel. Save and refresh: the west panel should now fully collapse, hiding its title.

  8. Use flex for the east panel

    Replace the east panel's width with flex:0.5

    Panels specifying widths and heights have a fixed value. Specifying flex makes the value proportional. By default, the center panel is flex 1. So setting the east panel to 0.5 means it will take up half the width of the center panel.

    Save and refresh. Is the east panel half the width of the center panel? Resize the window — are the center and east panels maintaining their relative widths?

    9. Optional

  9. Change weights to make west more important than east

    Change the panels' weights so that the west panel takes up the full height of the parent window and the north and south should take up all the remaining widths. The east takes should up the remaining space between the north and south panels.
