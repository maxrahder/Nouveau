Sencha Cmd is used in all facets of your project from scaffolding all the way to minifying and deploying your application to production.

These steps will guide you to the install process of the latest version of Sencha Cmd. In this class we will make use of the newest version of Sencha Cmd.

  1. In case you have Sencha SDK tools installed; remove them.
  2. Download the latest Sencha Cmd installer here
  3. Run the installer.
  4. Note the installation directory.

    By default, the installation path is (the version number could differ)
    Windows: C:\Users\Me\bin\Sencha\Cmd\3.*.*.***
    Mac OS X: ~/bin/Sencha/Cmd/3.*.*.***

  5. Sencha Cmd will be installed...
  6. Open your console and type: sencha, it should prompt all the sencha commands & options.

    In case you don't see anything, you will need to add sencha to the class path. See the instructions below.

  7. Check the Sencha Cmd version

    Type the following command in your console to check the Sencha Cmd version: sencha which

Windows users
Start > Control Panel > Performance and Maintenance > System
In your system properties click on the Advanced tab. Click the Environment Variables button. Edit the classpath variable or create a new variable called: classpath Add the following value and save:
C:\Users\Me\bin\Sencha\Cmd\3.*.*.***
Make sure you are entering the correct path and version number.

Mac users
Make hidden files visible. To achieve this, type the following command in your terminal:
defaults write com.apple.finder AppleShowAllFiles TRUE
killall Finder
(running these commands with FALSE will disable showing hidden files again) Now open ~.bash_profile.
Add the following commands and save the file:
export PATH=/Users/username/bin/Sencha/Cmd/3.*.*.***:$PATH
Make sure you are entering the correct path and version number.
