You can define validation rules on text fields by specifying the following properties:

Specify false to validate that the length of the entered value is greater than zero.

Sets the maximum (or minimum) length of input data.

A JavaScript RegExp object to be tested against the field value during validation.
Note that this validation rule will not produce a visible error unless the regexText property is also configured.

A custom validation function to be called during field validation. Return true if the value is valid. Otherwise return false.

Specifies whether this field should be validated immediately whenever a change in its value is detected. Defaults to true. If the validation results in a change in the field's validity, a validitychange event will be fired. If false, feedback will not be immediate. The form will still be validated before submitting if the clientValidation option to Ext.form.Basic-doAction is enabled, or if the field or form are validated manually.

A validation type name. Built-in validations include alpha, alphanum, email, and url. You can extend Ext.form.VTypes to define custom validators. One source of custom vtypes is the Sencha forum.
