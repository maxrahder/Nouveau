Web server

For this course we will make use of the built-in web server of Sencha Cmd. (Jetty 8.1.7). See how the Sencha Cmd built-in web server works.

XAMPP

When you prefer to run your own local (Apache) web server, we recommend using XAMPP, when no other webservers are installed, XAMPP should work on the fly after starting it!

Troubleshooting XAMPP & IIS

When IIS is installed, this one needs to be stopped.
Also kill all the SQL services, one of these is using port 80. If you can change the port to anything else; that should work too.

Tomcat

When using Apache Tomcat; copy the SenchaTraining files to the webapps folder. Make sure directory listings are enabled. By default this is disabled in Tomcat.

Open the file: conf/web.xml and add the following config: 

<init-param>
    <param-name>listings</param-name>
    <param-value>true</param-value>
</init-param>
