The controller knows that a search was requested, but not the specific user action that initiated the search. This decouples the what from the how. The code is simpler, and won't break because of some designer's change in how the action is initiated. 

Ext.define('MyApp.controller.TwitterController', {
    extend : 'Ext.app.Controller',
    
    views : [ 'TwitterGrid' ],
    stores : [ 'Tweets' ],
    models : [ 'Tweet' ],

    init : function() {
        this.control({
            'twittergrid' : {
                search : this.searchHandler
            }
        });
    },
    searchHandler : function(grid, searchString) {
        this.getTweetsStore().load({
            params : {
                q : searchString
            }
        });
    }
});
