Introduction

In this lab, you will start coding the YelpExtplorer application.

To see where you will end up at the end of the week you can visit it now in your browser.

This week we will build on the application until the complete application is coded, themed and deployed.

This lab assumes you've done the Create a Workspace lab.

Objectives

  • Inspect files of an Ext JS workspace
  • Generate initial application files
  • Verify initial application runs
  • Inspect files of an initial application

Steps

  1. Generate the YelpExtplorer application

    In the terminal window, navigate to SenchaTraining/ExtJS251/Labs/ext and generate the YelpExtplorer application by typing

    sencha generate app YelpExtplorer ../YelpExtplorer
  2. Run the application

    In your browser, open the initial application.

    You should see a single tab titled "Center Tab 1" in the center and a panel titled "west" on the left. The application doesn't do anything yet.

  3. Inspect the application

    In your IDE look at the SenchaTraining/ExtJS251/Labs/YelpExtplorer/ folder. You should see several files, organized according to Sencha's MVC file structure:

  4. Inspect index.html

    In your IDE open .../YelpExtplorer/index.html. Note that it references two JavaScript files: bootstrap.js and app.js.

    The bootstrap file contains classpath and alias mappings. Don't worry about the details of bootstrap.

  5. Inspect app.js

    In your IDE open .../YelpExtplorer/app.js.

    app.js is the starting point of an Ext JS application. To keep your logic separate and completely contained within the app folder, the code in app.js simply subclasses the Ext.app.Application defined in app/application.js.

  6. Inspect application.js

    In your IDE open .../YelpExtplorer/app/application.js.

    application.js is the starting point of your application. It's a normal class definition that extends Ext.app.Application. This is where you will specify the controllers used in the application.

  7. Inspect Main.js

    In your IDE open .../YelpExtplorer/app/controller/Main.js.

    This file contains the definition for a controller. Note that it extends Ext.app.Controller . Controllers coordinate events between data stores and components, but our controller doesn't do anything yet.

  8. Inspect Viewport.js

    In your IDE open .../YelpExtplorer/app/view/Viewport.js.

    This defines a viewport, using the fit layout. It creates an instance of app-main, which is defined in Main.js

  9. Inspect Main.js

    In your IDE open .../YelpExtplorer/app/view/Main.js.

    This view contains a layout:border container with two items: a panel on the west, and a tab panel in the center.

Conclusion

In this lab, you used Sencha Cmd to generate a new MVC application. This generated application contains demo code. As we write the YelpExtplorer app, we will be refactoring this starting point, adding new views models, stores and controllers.
