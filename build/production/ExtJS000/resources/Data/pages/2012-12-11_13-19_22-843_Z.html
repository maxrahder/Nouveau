Introduction

In this walkthrough, you will work with Sencha Architect 2 for Ext JS projects.

Together with your instructor, you will perform the following tasks:

  • Working with column templates

Steps

Promote the Business grid to a class

  1. Right-click on the Businesses grid in the Project Inspector.

  2. Choose the Promote to class menu item.

    Note: Your view may collapse in Design view, If this occurs, you can close Architect and reopen. It has no effect on the actual document when previewed.

    You will see a new entry in the Project Inspector for this component, with a link from the Viewport to the separate class.

  3. Under MyViewport, double-click the Business grid link to go directly to the new grid class.

  4. In the MyGridPanel, configure the following properties:

    • userClassName: Business

    • userAlias: bizgrid

  5. Click MyViewport, and go to Code view.

    Note: All the grid configuration code has been replaced by one line: xtype: 'bizgrid'

Convert MyViewport to border layout

  1. Click on MyViewport

  2. Set layout: border

    Note: In Code view, bizgrid is now in region: center.

Add a Custom Photo Component

  1. In the Toolbox, drag or double-click a "component" (generic component) to add it to MyViewport

  2. Set the following property to MyComponent. (the name of the just created component)

    • region: east

    Note: This will also automatically insert a width property of 150 pixels.

Promote the new component to a class

  1. Right-click on the component, choose Promote to class

  2. Change the following settings on the new class:

    • userClassName: BusinessDetail

    • userAlias: bizdetail

Add a photo to the BusinessDetail component

  1. Select the BusinessDetail component you just created

  2. Select the tpl property, and click on the → (arrow icon).

  3. Add the following code in the editor: 

    <div style="margin:1ex;padding:2ex">
<b>Name:</b><br>
{name}<br><br>
<b>Address:</b><br>
{full_address}<br><br>
<b>Photo:</b><br>
<img src={photo_url}>
</div>

Save and Preview the Application

Not much has changed, other than the addition of the “east” region. Nothing more is visible, at this time, because the component doesn’t have any data loaded that it can display. For that to happen, we have to listen for an event on the grid, then we can fill the custom photo component. That is the subject of our next lab, "Controller", we will populate the photo component with information from a selected grid row

Conclusion

We just created two new classes and added another template. We will soon see the data populating into the east region. Your patience will be rewarded.
